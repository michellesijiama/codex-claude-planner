# 实现任务：orchestrate.py（Claude–Codex 编排器）

请在当前仓库根目录创建一个文件 `orchestrate.py`。先阅读
`docs/superpowers/specs/2026-06-17-claude-codex-orchestrator-design.md` 了解完整设计，
然后严格按下面的契约实现。**只用 Python 3 标准库**（subprocess, json, pathlib, argparse,
datetime, re, sys, shutil, os）。代码要清晰、加中文注释、可直接运行。

## CLI

```
python3 orchestrate.py --goal "<目标>" [--repo .] [--model opus] [--max-plan-usd 1.0] [--max-escalations 2] [--dry-run]
```

## 必须实现的函数

1. `preflight(repo)`：确认 `claude`、`codex` 在 PATH（用 shutil.which），确认 repo 是 git 仓库
   （`git -C repo rev-parse --is-inside-work-tree`）。任何不满足则打印中文错误并 sys.exit(1)。

2. `claude_plan(goal, repo, model, max_usd) -> dict`：调用 Claude 生成计划。命令模板：
   ```
   claude -p <PROMPT> --output-format json --json-schema <PLAN_SCHEMA_JSON>
          --permission-mode plan --model <model> --max-budget-usd <max_usd>
   ```
   - PROMPT 要求 Claude 检视仓库、把目标拆成有序子任务，每个子任务给出 description 和一条
     可在 shell 直接运行的 check_command（测试/lint/build）。
   - PLAN_SCHEMA（结构化输出）：
     `{"type":"object","properties":{"subtasks":{"type":"array","items":{"type":"object",
       "properties":{"description":{"type":"string"},"check_command":{"type":"string"}},
       "required":["description","check_command"]}}},"required":["subtasks"]}`
   - claude 的 stdout 是一个 JSON 信封（含 result 字段）。要**健壮解析**：先 json.loads 整个 stdout，
     取其中的结构化对象；result 可能是字符串（需再 json.loads）或已是对象。提取出 subtasks 列表。
   - 给每个子任务补上 id（从 1 递增）、status="pending"、attempts=0、commit=null。

3. `codex_exec(subtask, repo) -> dict`：让 Codex 执行一个子任务。先把状态 schema 写到临时文件：
   `{"type":"object","properties":{"status":{"type":"string","enum":["DONE","BLOCKED"]},
     "summary":{"type":"string"}},"required":["status","summary"]}`
   命令模板：
   ```
   codex exec -s workspace-write -C <repo> --skip-git-repo-check
              --output-schema <STATUS_SCHEMA_FILE> -o <LAST_MSG_FILE> <PROMPT>
   ```
   - PROMPT：清楚说明这个子任务的 description，并告诉它完成后用 schema 汇报 status 和 summary。
   - 执行完读取 LAST_MSG_FILE（JSON），返回 {status, summary}。解析失败时返回
     {"status":"BLOCKED","summary":"无法解析 Codex 输出"}。

4. `run_check(check_command, repo) -> (bool, str)`：`subprocess.run(check_command, shell=True, cwd=repo,
   capture_output=True, text=True)`；返回 (returncode==0, 合并后的 stdout+stderr 尾部 4000 字符)。

5. `git_commit(repo, message)`：`git -C repo add -A` 然后 `git -C repo commit -m message`；
   返回短 commit hash（`git -C repo rev-parse --short HEAD`）。

6. `create_branch(repo, goal) -> str`：分支名 `orchestrator/<goal-slug>-<YYYYMMDD>`，slug 取 goal 前 30 字符、
   非字母数字转连字符。`git -C repo checkout -b <branch>`。返回分支名。

7. `claude_repair(subtask, last_error, repo, model) -> dict`：子任务连续失败后调用 Claude，
   命令同 claude_plan 风格，--json-schema 为
   `{"type":"object","properties":{"action":{"type":"string","enum":["retry","abort"]},
     "instructions":{"type":"string"}},"required":["action","instructions"]}`。
   返回 {action, instructions}。instructions 会作为追加说明再喂给 Codex 重试。

8. 状态文件：
   - `write_state(repo, state)`：把 state 写成 `mission.json`（缩进2）。
   - `render_markdown(repo, state)`：写人类可读的 `mission.md`，包含 goal、branch、每个子任务一行
     带状态 emoji（pending ⬜ / running ⏳ / done ✅ / failed ❌ / escalated ⚠️）。
   - 每次状态变化都调用这两个，让用户能 `tail -f mission.md`。

## 主循环（main）

```
parse args -> preflight -> plan = claude_plan(...) -> branch = create_branch(...)
state = {goal, branch, subtasks, log:[]}; write_state + render_markdown
for st in subtasks:
    st.status="running"; 刷新文件
    success=False
    for attempt in 1..2:
        st.attempts+=1
        res = codex_exec(st, repo)            # dry-run 模式下跳过真实调用，打印将执行的命令
        ok, tail = run_check(st.check_command, repo)
        if res.status=="DONE" and ok:
            st.commit = git_commit(repo, f"[orchestrator] 子任务{st.id}: {st.description}")
            st.status="done"; success=True; break
    if not success:
        escalations += 1
        if escalations > max_escalations:
            st.status="failed"; 记录日志; 打印"已达升级上限，交还用户"; break 整个循环
        decision = claude_repair(st, tail, repo, model)
        if decision.action=="abort": st.status="failed"; 记录; break
        else:  # retry：把 instructions 追加到子任务再来一轮（外层可用 while 重试一次）
            st.status="escalated"; 用新 instructions 再 codex_exec + run_check 一次；
            成功则 commit+done，否则 failed 并 break
    刷新文件
打印最终总结：完成几个、失败几个、分支名、提示用户 review 后自行合并。
```

## 要求
- `--dry-run` 时不真正调用 claude/codex/git 写操作，只打印将要执行的命令，便于测试。
- 所有面向用户的输出用中文。
- 顶部写一段 docstring 说明用法。
- 不要自动合并到 main。
